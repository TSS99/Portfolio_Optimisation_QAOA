# Limitations

This project is designed for learning, so it makes several simplifications.

- It uses mock financial data instead of live market data.
- It uses only three assets.
- It does not enforce a fixed budget.
- It does not require selecting exactly a certain number of assets.
- It uses a simulator instead of real quantum hardware.
- It does not compare QAOA against every classical optimization method.

These limits are useful for a first notebook because they keep the explanation focused. A more advanced version could add constraints, real data, and benchmarking.
